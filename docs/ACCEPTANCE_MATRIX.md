# 验收矩阵（ACCEPTANCE_MATRIX）

> 把《完整需求规格说明书》（`docs/PRODUCT_REQUIREMENTS.md`）的需求逐域映射到：所属 PR、验证方式、状态。作为全量验收（PR-09）的核对底稿，每个 PR 落地后回填。
> 状态枚举：`已验证` / `已实现` / `开发中` / `计划中` / `后续阶段`。

## 图例

- **验证方式**：`pytest`（后端自动化）、`vitest`（前端自动化）、`e2e`（docker-e2e 全栈脚本）、`manual`（人工对照参考图/操作）、`migration`（alembic 正反向 + check）。
