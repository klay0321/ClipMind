# 真实素材验收规范（REAL_MEDIA_ACCEPTANCE）

> 自 PR-05 起的长期制度。测试分**两层**：自动化回归层（CI，无密钥、可重复）与真实业务验收层
> （提交前本地手动，证明 AI 真的理解内容）。本文件不含任何真实路径、文件名、Key、Endpoint、
> 内网地址或客户信息。

## 1. 两层测试

### A. 自动化回归层（GitHub Actions / 本地 CI 等价）

允许且仅使用：

- `FakeProvider`（AI 分析）、`FakeEmbedding`、`Fake Script Parser`；
- 合成素材（FFmpeg `testsrc`/`smptebars` 等）；
- 固定测试数据、独立 pgvector 测试数据库；
- 无任何真实密钥、不下载真实模型、不读取真实视频/脚本。

证明（且仅证明）：

- 单元逻辑、API 契约、数据库约束与迁移、稳定分页、错误处理、降级路径；
- 端到端管线在 Fake 下水密闭环（创建→拆分→分析→索引→检索→重启持久化）；
- 可在无密钥环境完整、确定性地重复运行。

**不能**用自动化回归层证明"AI 真的理解了画面内容"。

### B. 真实业务验收层（每个相关 PR 提交前，本地手动）

凡涉及 **AI 理解 / 视频理解 / 检索 / 匹配 / 推荐 / 剪辑** 的 PR，提交前必须额外使用：

- 我的真实本地视频素材；
- 真实 MiMo（视觉/文本）、真实 E5 embedding；
- 真实拆镜头与真实关键帧；
- 真实脚本（或基于真实产品在本地生成、有实际营销逻辑的验收脚本）；
- **人工逐项内容核对**（看关键帧对照 AI 结果）。

必须证明：

- AI 描述与真实画面对应，不同内容产生不同结果（非固定模板）；
- 产品 / 场景 / 动作 / 风险来自真实画面，不编造；不确定时诚实降置信并标"需人工"；
- 搜索与脚本匹配的排序符合真实内容（如语义分对比矩阵的对角性）；
- 选择/锁定不被重匹配覆盖；剪辑清单引用真实 shot_id 与真实时间码。

**禁止**仅以 HTTP 200、字段非空、Celery 任务成功、数据库行数增加作为完成依据；
**禁止**以彩条 / `testsrc` / 单色 / 空视频 / 纯文字测试视频 / 全相同素材作为真实完成证据。

## 2. 本地配置与隔离（绝不提交真实值）

本地用现有 `.env` 或本地忽略文件保存真实配置；`.env.example` 只放空示例与说明。
约定的本地变量（示例，值仅存本地 `.env`）：

```text
CLIPMIND_REAL_MEDIA_ROOT=
RUN_REAL_MEDIA_ACCEPTANCE=1
RUN_REAL_PROVIDER_E2E=1
```

> 若系统已通过 `SourceDirectory` 记录真实素材目录，优先复用，不重复引入配置。
> 真实 MiMo 仅 docker 网络内可达时，真实 Provider 验收须在容器内运行（宿主直连可能超时）。

## 3. 真实验收产物保存位置（Git 忽略）

真实素材验收报告、关键帧、分析结果、脚本等一律保存在被 Git 忽略的目录：

```text
.local/real-media-acceptance/
```

**绝不**把真实视频、关键帧、代理、字幕、数据库 dump、分析结果、本地脚本全文、Key、Endpoint、
内网 IP、本机绝对路径提交到 Git。`.gitignore` 已忽略 `.local/`、`.env*`、`sample_media/**`、
`*.mp4/*.mov/*.webp`、`data/`。

## 4. 真实素材安全规则

- 真实素材**只读**：不重命名、不覆盖、不转码原文件、不删除；
- 派生（关键帧/代理/报告）只写数据卷或 `.local/`；
- 报告与日志中只用匿名编号（`product-video-01`…）、`asset_id`、时长、分辨率、画幅、镜头数；
- 不在任何提交物中出现真实文件名、客户/品牌私密信息、Key、Endpoint。

## 5. 与各 PR 的关系

- CI 的 Fake/合成端到端（含 `SCRIPT_GATE_A_E2E_OK` 等标志）保留并作为回归门禁；
- 但每个功能 PR 仍须在提交前完成 B 层真实验收，并把结论（符合/部分符合/不符合/无法判断）
  记录到 `.local/real-media-acceptance/` 的报告中（不提交）。
